# Mechanism Traces

## Purpose

Hold the main atlas body: documented traces where the proposed mechanism appears to become visible in development history, tool design, architecture evolution, or application essays.

## Material That Belongs Here

- internal operator traces
- tool-architecture traces
- application essay traces
- source-grounded summaries
- explicit uncertainty notes
- links to per-entry cards in `atlas/mechanism-trace/`

## Material That Must Not Go Here

- large numbers of weak entries
- cumulative case-building rhetoric
- sources included only by association

## Mechanism Review Standard

Each trace should be judged against:

- stuckness
- vaulting
- later pivot or return
- catalyst or topology fit
- rapid unlock or cascade
- structural centrality

The source does not need to use the final theory vocabulary directly.
What matters is whether the sequence can be reconstructed source-groundedly without inventing new material.

## Current Mechanism-Trace Body

The current atlas body is intentionally small and provisional.

It contains:

- internal operator provenance traces
- internal tool-architecture traces
- internal host-body and memory traces
- internal cross-repo fit traces

## Main Body Rule

The mechanism-trace tier is the main atlas body.

It should stay small.

Lead with the internal/operator/tool-architecture spine first.

Application essays belong after the operator, interaction, architecture, memory, and instrumentation lanes.

If a source is mainly:

- theory exposition
- formalism
- packaging
- tooling lens

then it belongs elsewhere.
